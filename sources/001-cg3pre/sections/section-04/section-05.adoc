=== 交通（徒歩道）モデルの応用スキーマ

徒歩道とは、公共測量標準図式における徒歩道及び庭園路（ただし、庭園路のうち、自動車ターミナル内の道路は、広場として取得するため、徒歩道には含まない）をいう。

ここで、公共測量標準図式における徒歩道とは、「道路縁及び軽車道に接続するもの、登山、観光等に利用されるもの、神社等主要な地点へ到達するもの、耕地の区画等の景観を表現するために必要なもの」であり、庭園路とは、「公園、住宅地等で自動車の通行を規制している道路及び工場等特定の敷地内の道路」である。


[[toc4_05_01]]
==== 交通（徒歩道）モデルのLOD

標準製品仕様書が対象とする交通（徒歩道）モデルのLODは、LOD0からLOD3までとする。

[[toc4_05_01_01]]
===== 交通（徒歩道）モデル（LOD0）

[[toc4_05_01_01_01]]
====== 交通（徒歩道）モデル（LOD0）の概要

交通（徒歩道）モデル（LOD0）では、徒歩道の形状を中心線により表現する。 +
このとき、属性granularityによって、徒歩道全体又は上下線単位なのか、あるいは車線単位なのかを表現する。 +
granularity=wayの場合は、徒歩道全体又は車道及び歩道の中心線を表現する。 +
さらに、中心線の粒度を詳細に区分するために属性fuctionを使用する。 +
granularity=laneの場合は、車線及び歩道の中心線を表現する。 +
標準製品仕様書は、原則として徒歩道全幅の中心線を採用する。ただし、ユースケースの必要に応じて他の粒度の中心線を採用できる。 +
また、数値地形図との互換性を保つために、徒歩道縁を採用できる。

徒歩道縁を採用する場合、公共測量法準図式に従う。（<<toc4_25_01_01,公共測量標準図式のためのLOD0の定義>>）

====== 交通（徒歩道）モデル（LOD2）の概要

交通（徒歩道）モデル（LOD2）では、徒歩道の形状を高さを持った面により表現する。 +
属性granularityによって、徒歩道内の区分の粒度を表現する。

交通（徒歩道）モデル（LOD2）は、「高さの取得方法」（<<tab-4-5-6>>）によりLOD2.0、LOD2.1及びLOD2.2に区分する。 +
また、「徒歩道内の区分」（<<tab-4-5-5>>）により属性granularityを区分する。 +
標準製品仕様は、原則としてLOD2.0とする。ただし、ユースケースの必要に応じて、LOD2.1及びLOD2.2を採用できる。
