[[toc4_03]]
=== 交通（道路）モデルの応用スキーマ

道路とは、一般交通の用に供する道であり、道路法第3条に示された道路の種類及び建築基準法第42条の定義を含む。


[[toc4_03_01]]
==== 交通（道路）モデルのLOD

標準製品仕様書が対象とする交通（道路）モデルのLODは、LOD0からLOD3までとする。

[[toc4_03_01_01]]
===== 交通（道路）モデル（LOD0）

[[toc4_03_01_01_01]]
====== 交通（道路）モデル（LOD0）の概要

交通（道路）モデル（LOD0）では、道路の形状を中心線により表現する。 +
このとき、属性granularityによって、道路全体又は上下線単位なのか、あるいは車線単位なのかを表現する。 +
granularity=wayの場合は、道路全体又は車道及び歩道の中心線を表現する。 +
さらに、中心線の粒度を詳細に区分するために属性fuctionを使用する。 +
granularity=laneの場合は、車線及び歩道の中心線を表現する。 +
標準製品仕様書は、原則として道路全幅の中心線を採用する。ただし、ユースケースの必要に応じて他の粒度の中心線を採用できる。 +
また、数値地形図との互換性を保つために、道路縁を採用できる。

道路縁を採用する場合、「<<gsi_ops,作業規程の準則 付録７ 公共測量標準図式>>」（以下、「公共測量標準図式」という）に従う。（<<toc4_25_01_01,公共測量標準図式のためのLOD0の定義>>） 

道路縁とは、「道路法第2条第1項に規定された道路にあっては道路構造令に定める歩道、自転車道、車道、中央帯、路肩、又は植樹帯等で構成される道路の部分で最も外側の線（植樹帯が最も外側にある場合には、当該植樹帯を除いた道路の最も外側の線をいう。）、道路法第2条第1項に規定する以外の道路にあってはこれに準ずる線」をいう。[<<gsi_ops,公共測量標準図式>>]

====== 交通（道路）モデル（LOD2）の概要

交通（道路）モデル（LOD2）では、道路の形状を高さを持った面により表現する。 +
属性granularityによって、道路内の区分の粒度を表現する。

標準製品仕様は、原則としてLOD2.0とする。ただし、ユースケースの必要に応じて、LOD2.1及びLOD2.2を採用できる。

===== Transportation（CityGML）

Transportationパッケージは、交通に関する地物型を定義する。

標準製品仕様では、道路（Road）、広場（Square）、徒歩道（Track）、鉄道（Railway）及び航路（Waterway）を定義する。

これらは、道路を構成する歩道や車道のような通行可能な領域（TrafficArea）と、道路における路肩のように、これを補助する役割をもつ領域（AuxiliaryTrafficArea）の集まりとして構成できる。
