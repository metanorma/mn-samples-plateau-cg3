[[tocM_02]]
=== トンネルの記述とLOD


[[tocM_02_01]]
==== トンネルモデル

トンネルとは、上方を含め周辺が地山や他の構造物で覆われている、交通、輸送等に供する構造物である。[道路基盤地図情報（整備促進版）製品仕様書（案）] +
標準製品仕様書では、その設置の形態から、ボックスカルバート、シェッド、シェルター及び地下横断歩道もトンネルに含める。

標準製品仕様書では、トンネルの表現に必要な地物型等をCityGML及びi-URから抽出し、LODごとに、含むべき地物型、各地物型の空間属性の型、取得基準、取得方法及び補足を定めた「トンネルモデル」を定義する。

トンネルモデルのLODは、LOD0からLOD3までを対象とする。このとき、LODには外部と内部がそれぞれ存在する。

[[fig-M-1]]
.トンネルモデルの概要
image::../../sources/images/anx_M_001.png[]

* トンネルモデル・外部（LOD0）は、トンネルの形状を、トンネルの上方からの正射影の外周により囲まれた平面により表現する。
* トンネルモデル・外部（LOD1）は、トンネルの形状を、トンネルの上方からの正射影の外周を一律の高さで上向きに押し上げた立体（箱モデル）により表現する。
* トンネルモデル・外部（LOD2）は、トンネルの形状を、立体により表現し、立体を構成する各境界面を地物として取得する。
* トンネルモデル・外部（LOD3）は、トンネルの形状を、構造部材の外形を表す面により構成された多曲面（面の集まり）として表現する。
* トンネルモデル・内部（LOD0）は、トンネル内部の形状を、外周を表す面又は線により表現する。
* トンネルモデル・内部（LOD1）は、トンネル内部の形状を、外周を表す面を一律の高さで上向きに押し出した立体により表現する。
* トンネルモデル・内部（LOD2）は、トンネル内部の形状を、床や天井の形状を含む立体として表現し、その境界面を天井面、内壁面、床面等に区分する。
* トンネルモデル・内部（LOD3）は、トンネル内部の形状を床や天井の形状を含む立体として表現し、その境界面を天井面、内壁面、床面等に区分するとともに、構造部材の外形を多曲面として表現する。


[[tocM_02_02]]
==== 使用可能な地物型とLOD

トンネルモデルは、LODごとに、使用すべき地物型やその空間属性が異なる。


[requirement]
.トンネル記述のLOD対応条件
====
[%metadata]
identifier:: /req/tun/1
subject:: 妥当なトンネルデータセットの要件
description:: トンネルの記述には、指定されたLODに対応する地物型及び空間属性を使用する。
====

トンネルは、CityGMLのTunnelモジュールに定義されたtun:Tunnelを用いて記述する。 +
トンネル、これを構成する屋根及び壁、開口部（窓や扉）、バルコニー等の付属物、部屋、構造部材等は、LODに応じた空間属性をもつ。

* 外部LOD
** 外部（LOD0）では、tun:Tunnelを使用し、トンネルの幾何形状を平面又は線により記述する。
** 外部（LOD1）では、tun:Tunnelを使用し、トンネルの幾何形状を立体で記述する。
** 外部（LOD2）では、tun:Tunnelを使用し、トンネルの幾何形状を立体又は多曲面で記述する。また、立体を構成する境界面又は多曲面を構成する面を、屋根面や外壁面等の地物型を用いて記述する。屋根面や外壁面はcon:AbstractConstructionSurfaceとして定義されている。con:AbstractConstructionSurfaceは抽象地物であり、実装にはこれを継承する地物型（例：屋根面はcon:RoofSurface）を使用する。また、扉面（con:DoorSurface）や窓面（con:WindowSurface）を用いることもできる。
** 外部（LOD3）では、tun:Tunnelを使用し、トンネルの幾何形状を立体又は多曲面で記述する。また、トンネルは構造部材（tun:TunnelConstructiveElement）の集まりとして構成され、トンネルの立体を構成する境界面又は多曲面を構成する面は、構造部材を構成する面に一致する。また、扉（con:Door）及び窓（con:Window）を含めることができる。
* 内部LOD
** 内部（LOD0）では、tun:Tunnelを使用し、トンネルを内部空間（tun:HollowSpace）の集まりとして記述する。内部空間の幾何形状は、平面又は線で記述する。
** 内部（LOD1）では、tun:Tunnelを使用し、トンネルを内部空間（tun:HollowSpace）の集まりとして記述する。内部空間の幾何形状は、立体で記述する。
** 内部（LOD2）では、tun:Tunnelを使用し、トンネルを内部空間（tun:HollowSpace）の集まりとして記述する。内部空間の幾何形状は、立体で記述する。また、立体の境界面を天井面や内壁等の地物型を用いて記述する。天井面や内壁面はcon:AbstractConstructionSurfaceとして定義されている。con:AbstractConstructionSurfaceは抽象地物であり、実装にはこれを継承する地物型（例：天井面はcon:CeilingSurface）を使用する。また、扉面（con:DoorSurface）や窓面（con:WindowSurface）を用いることもできる。
** 内部（LOD3）では、tun:Tunnelを使用し、トンネルを内部空間（tun:HollowSpace）及び構造部材（tun:TunnelConstructiveElement）の集まりとして記述する。また、扉（con:Door）及び窓（con:Window）を含めることができる。

各LODでは、tun:TunnelPartを使用できる。tun:TunnelPartは、一つのトンネルを二つ以上の部分に分けたトンネルの区間であり、tun:Tunnelと同様の属性をもつことができる。

例えば、トンネルが区間により構造が異なっていたり、データ取得に使用する原典資料が異なっていたりする場合に、トンネルを複数の区間に区切り、区切ったトンネルの部分（tun:TunnelPart）ごとに必要な属性を与えることができる。

また、トンネルモデル・外部（LOD1）において、トンネル内に傾斜があり、入口と出口の標高が異なる場合には、複数の区間に分けて記述することで、より実際に近い形状で表現することができる。

ただし、このとき、[underline]##tun:Tunnelを構成するtun:TunnelPartとは、一体的な建築物（tun:TunnelPart同士が互いに接して一棟の建築物を構成する）でなければならない。##

また、各LODではトンネルに付属する階段や管理用通路といった設備を、tun:TunnelInstallationを用いて記述できる。なお、tun:TunnelPartはトンネルであるが、tun:TunnelInstallationはトンネルではなく、トンネルの付属物である。外部LODで使用する場合、tun:TunnelInstallationは屋外の付属物であり、内部LODで使用する場合は屋内の付属物となる。

さらに、内部LODでは、可動設備（tun:TunnelFurniture）を含めることができる。

==== トンネルモデルのLOD定義

トンネルモデルは、LODごとに、含むべき地物やその取得基準を定めている。

[[tocM_02_03_01]]
===== 外部LOD0

トンネルモデル・外部（LOD0）は、含むべき地物により、LOD0.0及びLOD0.1に細分される。

LOD0.0はトンネルの外周を面により表現する。 +
LOD0.1は、接地面の正射影の外周か屋根面の正射影の外周かを区別して取得する。

標準製品仕様書では、原則としてLOD0.0を採用する。ただし、ユースケースの必要に応じてLOD0.1を採用できる。

[[tocM_02_03_02]]
===== 外部LOD1

トンネルモデル・外部（LOD1）は、含むべき地物によりLOD1.0及びLOD1.1に細分される。

LOD1.0は、core:lod0MultiSurfaceが示すトンネルの外周を示す平面（又はcore:lod0MultiCurveが示すトンネルの外周を閉じて作成した平面）を、一定の高さで上向きに押し出した角柱として表現したものである。 +
LOD1.1は、LOD1.0の立体を構成する境界面を地物に区分する。

標準製品仕様書は、原則としてLOD1.0を採用する。ただし、ユースケースの必要に応じてLOD1.1を採用できる。

[[tocM_02_03_03]]
===== 外部LOD2

トンネルモデル・外部（LOD2）は、測量による取得を前提とした区分である。 +
トンネルの外形を立体として表現する。トンネルの立体を構成する面は、屋根面（con:RoofSurface）、外壁面（con:WallSurface）及び底面（con:GroundSurface）に区分する。

また、ユースケースの必要に応じて、開口部の境界面（con:DoorSurface及びcon:WindowSurface）を使用することができる。また、さらに付属物（tun:TunnelInstallation）を表現することもできる。

===== 外部LOD3

トンネルモデル・外部（LOD3）は、含むべき地物によりLOD3.0及びLOD3.1に細分される。 +
トンネルモデル（LOD3）は、CIMモデルやCADデータからの変換による作成を前提とした区分である。

LOD3.0は、トンネルを構造部材（tun:TunnelConstructiveElement）の集まりとして表現し、トンネルの形状を、構造部材を構成する面を集めた多曲面として表現する。 +
LOD3.1は、LOD3.0におけるトンネルの幾何形状を構成する各面を、境界面を表す地物に区分することができる。

標準製品仕様書は、原則としてLOD3.0を採用する。ただし、ユースケースの必要に応じてLOD3.1を採用できる。

[[tocM_02_03_07]]
===== 内部LOD2

トンネルモデル・内部（LOD2）は、トンネルの内部空間の形状を、床や天井の形状を再現した立体として表現する。 +
トンネルモデル・内部（LOD2）は含むべき地物やその大きさにより、LOD2.0、LOD2.1及びLOD2.2に細分される。トンネルモデル・内部（LOD2）は、測量による取得を前提とした区分である。

LOD2.0は、橋梁の内部空間を立体として取得し、その境界面を地物に区分する。 +
LOD2.1は、橋梁の内部空間を立体として取得し、その境界面を地物に区分する。さらに規模の大きなトンネル内部の付属物を取得する。 +
LOD2.2は、橋梁の内部空間を立体として取得し、その境界面を地物に区分する。さらにトンネル内部の付属物を取得する。可動設備を取得することもできる。

標準製品仕様書は、原則としてLOD2.0を採用する。ただし、ユースケースの必要に応じてLOD2.1及びLOD2.2を採用できる。

トンネルモデル・内部（LOD3）は、含むべき地物によりLOD3.0及びLOD3.1に細分される。 +
その他の構造物モデル（LOD3）は、CIMモデルやCADデータからの変換による作成を前提とした区分である。

LOD3.0は、トンネルを、内部空間及び構造部材の集まりとして表現し、内部空間は立体、構造部材は多曲面として記述する。 +
LOD3.1は、LOD3.0における内部空間を構成する各面を、地物に区分することができる。
