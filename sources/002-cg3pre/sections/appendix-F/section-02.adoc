[[tocF_02]]
=== 徒歩道の記述とLOD


[[tocF_02_01]]
==== 交通（徒歩道）モデル

徒歩道とは、公共測量標準図式における徒歩道及び庭園路（ただし、庭園路のうち、自動車ターミナル内の道路は、徒歩道として取得するため、徒歩道には含まない）をいう。 +
徒歩道の記述には、CityGMLのTransportationモジュールに定義されたtran:Trackを使用する。

標準製品仕様書では、徒歩道の表現に必要な地物型等をCityGML及びi-URから抽出し、LODごとに、含むべき地物型、各地物型の空間属性の型、取得基準、取得方法及び補足を定めた「交通（徒歩道）モデル」を定義する。

交通（徒歩道）モデルのLODは、LOD0からLOD3までを対象とする。このとき、LODには外部・内部の区別はない。

[[fig-F-1]]
.交通（徒歩道）モデルの概要
image::../../sources/images/anx_F_001.png[]

交通（道路）モデルと同様、LOD0は徒歩道の形状をネットワークとして表現する。 +
また、LOD1、LOD2及びLOD3では、徒歩道の形状は路面を表す。加えて、これらのLODでは、徒歩道の空間を立体として表現することができる。 +
すべてのLODにおいて、路面上に存在するマンホールや損傷を、地物として区別して表現することができる。 +
さらに、すべてのLODにおいて、道路を「区間」や「交差点」に区切ることができる。


[[tocF_02_02]]
==== 使用可能な地物型とLOD

交通（徒歩道）モデルは、LODごとに、使用すべき地物型やその空間属性が異なる。


[requirement]
.徒歩道記述のLOD対応条件
====
[%metadata]
identifier:: /req/trk/1
subject:: 妥当な交通（徒歩道）データセットの要件
description:: 徒歩道の記述には、指定されたLODに対応する地物型及び空間属性を使用する。
====

徒歩道の記述には、tran:Trackを用いる。CityGML3.0において、tran:Trackは、徒歩道の空間を表す。 +
の空間は、交通のために使用される交通空間（tran:TrafficSpace）、交通のために使用されない空間（tran:AuxiliaryTrafficSpace）に分かれる。また、交通空間上空の障害物のない空間（tran:ClearanceSpace）に分けることもできる。 +
tran:TrafficSpace及びtran:AuxiliaryTrafficSpaceが表す空間の底面は、路面となる。それぞれに対応する路面は、交通領域（tran:TrafficArea）及び交通補助領域（tran:AuxiliaryTrafficArea）である。 +
また、これらの路面上に存在するマンホールの蓋や損傷は、穴の面（tran:HoleSurface）として区別することができる。 +
さらに、複数の空間を、区間（tran:Section）や交差点（tran:Intersection）としてグループ化できる。

徒歩道の記述に使用する各地物型は、空間属性を持つものと持たないものがある。空間属性を持つ場合、LODに応じた空間属性を持つ。

* LOD0では、tran:Trackに含まれるtran:TrafficSpaceの境界となるtran:TrafficAreaを使用し、徒歩道の形状を線で記述する。
* LOD1では、tran:Trackに含まれるtran:TrafficSpaceの境界となるtran:TrafficAreaを使用し、徒歩道の形状を面で記述する。また、必要に応じてtran:AuxiliaryTrafficSpaceの境界となるtran:AuxiliaryTrafficAreaを区分することができる。
* LOD2では、tran:Trackに含まれるtran:TrafficSpaceの境界となるtran:TrafficArea及びtran:AuxiliaryTrafficSpaceの境界となるtran:AuxiliaryTrafficAreaを使用し、徒歩道の形状を、高さを持つ面で記述する。
* LOD3では、tran:Trackに含まれるtran:TrafficSpaceの境界となるtran:TrafficArea及びtran:AuxiliaryTrafficSpaceの境界となるtran:AuxiliaryTrafficAreaを使用し、徒歩道の形状を、高さを持つ面で記述する。このとき、LOD2よりもさらに詳細な形状が表現される。

tran:Trackは空間属性を持たず地物のコンテナとして機能する。このとき、徒歩道の形状はこれに含まれる地物の空間属性により表現する。 +
各LODでは、tran:Section及びtran:Intersectionを使用できる。これらは、tran:TrafficSpace及びtran:AuxiliaryTrafficSpaceの集まりとなる。tran:Section（区間）及びtran:Intersectionも空間属性は持たない。各LODにおいて、tran:TrafficSpace及びtran:AuxiliaryTrafficSpaceは、空間属性を省略できる。 +
さらに、各LODでは、路面上に存在するマンホールの蓋や損傷を表す範囲をtran:HoleSurfaceとして記述することができる。

[[tocF_02_03]]
==== 交通（徒歩道）モデルのLOD定義

交通（徒歩道）モデルは、LODごとに、含むべき地物やその取得基準を定めている。


[[tocF_02_03_01]]
===== LOD0

交通（徒歩道）モデル（LOD0）は、徒歩道の形状を、高さを持たない中心線として記述する。中心線の記述には、tran:TrafficAreaの空間属性（core:lod0MultiCurve）を使用する。 +
このとき、中心線は、道路、上り線と下り線、車線、歩道のいずれかとなる。中心線の粒度は、<<tab-F-2>>に示す主題属性を用いて区分する。

| 車線の中心線 | lane | 車線

[[tocF_02_03_02]]
===== LOD1

交通（徒歩道）モデル（LOD1）は、徒歩道の形状を、高さを持たない多曲面として記述する。多曲面の記述には、tran:TrafficAreaの空間属性（core:lod1MultiSurface）を使用する。 +
このとき、多曲面は道路全幅、上り線/下り線、車線又は歩道のいずれかとなる。

[[tocF_02_03_03]]
===== LOD2

交通（徒歩道）モデル（LOD2）は、徒歩道の形状を、高さを持つ多曲面として記述する。 +
多曲面の記述には、tran:TrafficAreaの空間属性（core:lod2MultiSurface）及びtran:AuxiliaryTrafficAreaの空間属性（core:lod2MultiSurface）を使用する。

交通（徒歩道）モデル（LOD2）は、徒歩道内の区分と高さの取得方法により、LOD2.0、LOD2.1及びLOD2.2に細分される。<<tab-F-5>>及び<<tab-F-6>>に、細分したLOD2の概要を示す。

// 交通（徒歩道）モデルにおける属性granularityによる「徒歩道内の区分」
[[tab-F-5]]
include::../../../common/sections/com_trk_Category_lod2.adoc[]

// 交通（徒歩道）モデルにおけるLOD2.0、LOD2.1及びLOD2.2の「高さの取得方法」
[[tab-F-6]]
include::../../../common/sections/com_trk_Category_height_lod2.adoc[]

LOD1では徒歩道オブジェクトは高さを持たないため、立体交差や歩道と車道との間の高低差を表現できない。これらが必要な場合には、高さを持つLOD2又はLOD3が必要となる。

// 交通（徒歩道）モデル（LOD2）の取得イメージ（徒歩道内の区分）
[[tab-F-7]]
include::../../../common/sections/com_trk_Acquire_lod2_category.adoc[]

LOD2において最も粗い「徒歩道内の区分」では、徒歩道を車道部、車道交差部、島及び歩道部に区分する。この区分はLOD2.0に適用され、この場合、tran:TrafficSpaceの属性tran:granularityの値は、“way”となる。

また、LOD2.0において、車線を区分する場合、tran:TrafficSpaceの属性tran:granularityの値は、“lane”となる。 +
OD2.1では、LOD2.0の区分に加え、歩道部のうち歩道上の植栽を区分する。すなわち、歩道においても、通行の用に供される部分を明確に区分できる。このとき、車道部も車線に区分され、tran:TrafficSpaceの属性tran:granularityの値は、“lane”となる。

LOD2.2では、徒歩道の横断構成要素をさらに細分する。車道部のうち、路肩や停車帯等を区別したり、歩道を歩道や自転車歩行車道等に区別したりできる。LOD2.2における横断構成要素の取得基準の区分はユースケースごとに決定できる。

// 交通（徒歩道）モデル（LOD2）の取得イメージ（高さの取得方法）
[[tab-F-8]]
include::../../../common/sections/com_trk_Acquire_lod2_height.adoc[]

LOD2.0の「高さの表現」では、高架橋、立体交差等、徒歩道全体の上下の階層構造を区別できる。

LOD2.1では、道路内の縁石による高さの差を表現する。この高さの差は、縁石によりマウントアップされた歩道と車道との高さの差や車道内の分離帯や交通島の高さの差（概ね15㎝程度）とする。横断歩道への接続や車両の出入り等の目的で歩道に設けられた、歩道の切り下げ部では緩やかに車道の高さに擦り付ける（歩道切り下げ部と車道との間に生じる高さの差の表現は行わない）。

さらに、LOD2.2では、歩道切り下げ部と車道との高さの差（概ね2㎝程度）の表現を行う。なお、LOD2.2における高さの取得基準の下限値はユースケースにより決定できる。

なお、LOD2.0及びLOD2.1は、航空写真等上空から取得したデータの利用を前提とした区分である。このとき、トンネル内や高架橋の下部等の遮蔽部は上空から取得したデータでは作成することができない。そのため、他の資料による補完又は、推定によるデータ作成をする必要がある。LOD2.2は、MMS（Mobile Mapping System）により取得した点群や画像等のデータの利用を前提とした区分である。

データ集合に、航空写真等による図化、他の資料による補完、推定による作図というように、作成方法が異なる交通（徒歩道）オブジェクトが混在する場合は、データ品質属性（<<tocF_03_02_01>>）を使用し、個々の交通（徒歩道）オブジェクトの品質情報を記述することで、それぞれの作成方法を明示できる。


[[tocF_02_03_04]]
===== LOD3

交通（徒歩道）モデル（LOD3）は、徒歩道の形状を、高さを持つ多曲面として記述する。

多曲面の記述には、tran:TrafficAreaの空間属性（core:lod3MultiSurface）及びtran:AuxiliaryTrafficAreaの空間属性（core:lod3MultiSurface）を使用する。

LOD3は、徒歩道の形状をさらに詳細に表現することができる。例えば、徒歩道の横断構成要素をさらに細分したり、徒歩道の表面の凹凸を表現したりできる。

LOD3は、徒歩道の詳細設計等で作成される3次元モデル成果物の利用を想定した区分である。

===== LODの対照表

CityGML2.0とCityGML3.0では、LOD定義が異なることから、CityGML2.0に準拠した標準製品仕様書第5.1版と、CityGML3.0に準拠した本応用スキーマにおけるLODの細分の定義は異なる。

細分の対応は、<<tocD_02_03_05,交通（道路）モデル>>を参照のこと。
