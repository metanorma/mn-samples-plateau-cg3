[[tocE_02]]
=== 鉄道の記述とLOD


[[tocE_02_01]]
==== 交通（鉄道）モデル

鉄道とは、人と物を迅速かつ大量に輸送するため、レールを敷いた専用の通路を用い、その上を車両が円滑に行き来できるように整備された一切の設備とシステムの集合体である。[一般社団法人日本民営鉄道協会] +
標準製品仕様書では、鉄道事業法及び軌道法に基づいて敷設された線路を指し、以下を含む。

* 普通鉄道：鉄道事業法又は軌道法に基づいて運行されている鉄道で、特殊軌道及び索道を除いたもの[<<gsi_ops,付録７ 公共測量標準図式>>]
* 地下鉄：地方公共団体及び東京地下鉄（株）等が管理する地下高速鉄道[<<gsi_ops,付録７ 公共測量標準図式>>]
* 路面電車：道路上に線路を敷設した鉄道で、主として路面上から直接乗り降りできる車両が運行される鉄道[<<gsi_ops,付録７ 公共測量標準図式>>]
* モノレール：車両が一本の軌道桁に跨座し、又は懸垂して走行するもの
* 特殊鉄道：鋼索鉄道、普通鉄道と接続しない工場等特定の地区内の軌道及び採鉱（石）地と工場等を結ぶ専用軌道[<<gsi_ops,付録７ 公共測量標準図式>>]
* 索道：空中ケーブル、スキーリフト、ベルトコンベヤー及びこれらに類するもの[<<gsi_ops,付録７ 公共測量標準図式>>]

なお、線路とは、列車又は車両を走らせるための通路であって，軌道及びこれを支持するために必要な路盤、構造物を包含する地帯をいう。[<<jis_e_1001>>]

標準製品仕様書では、鉄道の表現に必要な地物型等をCityGML及びi-URから抽出し、LODごとに、含むべき地物型、各地物型の空間属性の型、取得基準、取得方法及び補足を定めた「交通（鉄道）モデル」を定義する。

交通（鉄道）モデルのLODは、LOD0からLOD3までを対象とする。

[[fig-E-1]]
.交通（鉄道）モデルのLODの考え方
image::../../sources/images/anx_E_001.png[]

交通（鉄道）モデル（LOD0）は鉄道を線として表現するが、このときの線は、ネットワーク（軌道中心線又は鉄道中心線）とする。ネットワークデータとして記述することでマクロな人や物の流れを表現・分析可能となり、公共交通網の見直しや公共施設の再配置等に利用することを想定している。

交通（道路）モデルと同様、LOD0は鉄道の形状をネットワークとして表現する。

また、LOD1、LOD2及びLOD3では、鉄道の形状は路面を表す。加えて、これらのLODでは、鉄道の空間を立体として表現することができる。

すべてのLODにおいて、鉄道上に存在する排水溝等を、地物として区別して表現することができる。

さらに、すべてのLODにおいて、鉄道を「区間」や「交差部」に区切ることができる。


[[tocE_02_02]]
==== 使用可能な地物型とLOD

交通（鉄道）モデルは、LODごとに、使用すべき地物型やその空間属性が異なる。


[requirement]
.鉄道記述のLOD対応条件
====
[%metadata]
identifier:: /req/rwy/1
subject:: 妥当な交通（鉄道）データセットの要件
description:: 鉄道の記述には、指定されたLODに対応する地物型及び空間属性を使用する。
====

鉄道の記述には、tran:Railwayを用いる。CityGML3.0において、tran:Railwayは、鉄道の空間を表す。 +
この空間は、交通のために使用される交通空間（tran:TrafficSpace）、交通のために使用されない空間（tran:AuxiliaryTrafficSpace）に分かれる。また、交通空間上空の障害物のない空間（tran:ClearanceSpace）に分けることもできる。

tran:TrafficSpace及びtran:AuxiliaryTrafficSpaceが表す空間の底面は、鉄道敷となる。それぞれに対応する底面は、交通領域（tran:TrafficArea）及び交通補助領域（tran:AuxiliaryTrafficArea）である。

また、これらの面上に存在する排水溝や水路等の溝の蓋は、穴の面（tran:HoleSurface）として区別することができる。

さらに、複数の空間を、区間（tran:Section）や交差点（tran:Intersection）としてグループ化できる。

交通（鉄道）モデルの記述に使用する各地物型は、空間属性を持つものと持たないものがある。空間属性を持つ場合、LODに応じた空間属性を持つ。

* LOD0では、tran:Railwayに含まれるtran:TrafficSpaceの境界となるtran:TrafficAreaを使用し、鉄道の形状を線で記述する。
* LOD1では、tran:Railwayに含まれるtran:TrafficSpaceの境界となるtran:TrafficAreaを使用し、鉄道の形状を面で記述する。
* LOD2では、tran:Railwayに含まれるtran:TrafficSpaceの境界となるtran:TrafficArea及びtran:AuxiliaryTrafficSpaceの境界となるtran:AuxiliaryTrafficAreaを使用し、鉄道の形状を、高さを持つ面で記述する。このときの高さは、軌道中心線の高さとする。
* LOD3では、tran:Railwayに含まれるtran:TrafficSpaceの境界となるtran:TrafficArea及びtran:AuxiliaryTrafficSpaceの境界となるtran:AuxiliaryTrafficAreaを使用し、鉄道の形状を、高さを持つ面で記述する。このとき、LOD2よりもさらに詳細な形状が表現される。

tran:Railwayは空間属性を持たず地物のコンテナとして機能する。このとき、鉄道形状はこれに含まれる地物の空間属性により表現する。

各LODでは、tran:Section及びtran:Intersectionを使用できる。これらは、tran:TrafficSpace及びtran:AuxiliaryTrafficSpaceの集まりとなる。tran:Section（区間）及びtran:Intersectionも空間属性は持たない。

各LODにおいて、tran:TrafficSpace及びtran:AuxiliaryTrafficSpaceは、空間属性を省略できる。

さらに、各LODでは、軌道敷に存在する溝の蓋や溝の範囲をtran:HoleSurfaceとして記述することができる。

==== 交通（鉄道）モデルのLOD定義

交通（鉄道）モデルは、LODごとに、含むべき地物やその取得基準を定めている。


[[tocE_02_03_01]]
===== LOD0

交通（鉄道）モデル（LOD0）は、鉄道の形状を、高さを持たない中心線として記述する。

中心線の記述には、tran:TrafficAreaの空間属性（core:lod0MultiCurve）を使用する。このとき、中心線は、軌道中心線又は鉄道中心線のいずれかとなる。

[[tocE_02_03_02]]
===== LOD1

交通（鉄道）モデル（LOD1）は、鉄道の形状を、高さを持たない多曲面として記述する。 +
多曲面の記述には、tran:TrafficAreaの空間属性（core:lod1MultiSurface）を使用する。

交通（鉄道）モデル（LOD1）は、取得対象の異なるLOD1.0とLOD1.1に細分される。

* LOD1.0：普通鉄道、地下鉄、路面鉄道及び特殊軌道
* LOD1.1：普通鉄道、地下鉄地上部、路面鉄道、特殊鉄道、地下鉄地上部、路面鉄道、特殊鉄道、トンネル内の鉄道、地下鉄地下部

標準作業手順書では、原則としてLOD1.0を採用する。ただし、ユースケースの必要に応じて、トンネル内の鉄道や地下鉄地下部の鉄道を表現するためにLOD1.1を採用することもできる。

[[tocE_02_03_03]]
===== LOD2

交通（鉄道）モデル（LOD2）は、鉄道の形状を、高さを持つ多曲面として記述する。 +
多曲面の記述には、tran:TrafficAreaの空間属性（core:lod2MultiSurface）及びtran:AuxiliaryTrafficAreaの空間属性（core:lod2MultiSurface）を使用する。

さらに、交通（鉄道）モデル（LOD2）では、i-URにより拡張したuro:lod2networkを用いて、軌道中心線を記述する。

交通（鉄道）モデル（LOD2）は、鉄道内の区分と高さの取得方法により、LOD2.0、LOD2.1及びLOD2.2に細分される。<<tab-E-5>>及び<<tab-E-6>>に、細分したLOD2の概要を示す。 +
標準製品仕様は、原則としてLOD2.0を採用する。ただし、ユースケースの必要に応じて、LOD2.1又はLOD2.2を採用できる。

LOD2.0の「高さの表現」では、軌道中心線の各点に標高を与えるとともに、道床に軌道中心線上の標高を与える。これにより、高架橋、立体交差等、鉄道全体の上下の階層構造を区別できる。

LOD2.1では、軌道中心線の各点に標高を与えるとともに、レールの横断方向に存在する15㎝以上の高さの差を取得する。 +
さらに、LOD2.2では、軌道中心線の各点に標高を与えるとともに、レールの横断方向に存在する15㎝未満の高さの差を取得する。高さの差を取得する閾値は、ユースケースの必要に応じて定めることができる。

データ集合に、航空写真等による図化、他の資料による補完、推定による作図というように、作成方法が異なる鉄道オブジェクトが混在する場合は、データ品質属性（<<tocE_03_02_01>>）を使用し、個々の鉄道オブジェクトの品質情報を記述することで、それぞれの作成方法を明示できる。

[[tocE_02_03_04]]
===== LOD3

交通（鉄道）モデル（LOD3）は、鉄道の形状を、高さを持つ多曲面として記述する。

多曲面の記述には、tran:TrafficAreaの空間属性（core:lod3MultiSurface）及びtran:AuxiliaryTrafficAreaの空間属性（core:lod3MultiSurface）を使用する。 +
さらに、交通（鉄道）モデル（LOD3）では、i-URにより拡張したuro:lod3networkを用いて、軌道中心線を記述する。

LOD3は、鉄道の形状をさらに詳細に表現することができる。例えば、鉄道の横断構成要素をさらに細分したり、レールの横断方向の高低差をより詳細に表現したりできる。 +
LOD3は、鉄道の詳細設計等で作成される3次元モデル成果物の利用を想定した区分である。
