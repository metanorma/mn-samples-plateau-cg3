[[tocD_02]]
=== 道路の記述とLOD


[[tocD_02_01]]
==== 交通（道路）モデル

道路とは、一般交通の用に供する道であり、道路法第3条に示された道路の種類及び建築基準法第42条の定義を含む。

標準製品仕様書では、道路の表現に必要な地物型等をCityGML及びi-URから抽出し、LODごとに、含むべき地物型、各地物型の空間属性の型、取得基準、取得方法及び補足を定めた「交通（道路）モデル」を定義する。

道路モデルのLODは、LOD0からLOD3までを対象とする。このとき、LODには外部・内部の区別はない。

[[fig-D-1]]
.交通（道路）モデルの概要
image::../../sources/images/anx_D_001.png[]

* 交通（道路）モデル（LOD0）は、道路の形状を多曲線（線の集まり）により表現する。
* 交通（道路）モデル（LOD1）は、道路の形状を、高さを持たない多曲面（面の集まり）により表現する。
* 交通（道路）モデル（LOD2）は、道路の形状を、高さを持つ多曲面により表現する。
* 交通（道路）モデル（LOD3）は、道路の形状を、高さを持つ多曲面により表現する。このとき、LOD2よりもさらに詳細な形状が表現される。

LOD1、LOD2及びLOD3では、道路の形状は路面を表す。加えて、これらのLODでは、道路の空間を立体として表現することができる。 +
すべてのLODにおいて、路面上に存在するマンホールや損傷を、地物として区別して表現することができる。 +
さらに、すべてのLODにおいて、道路を「区間」や「交差点」に区切ることができる。

LOD0における道路は線、LOD1以降では面として表現する。また、その座標はLOD0及びLOD1は高さを持たず、LOD2及びLOD3では高さを持つ。このとき、LOD1及びLOD2では、交通（道路）オブジェクトを3次元の地形データに重畳することで3次元的に表示して利用することを想定している。

LOD0は道路をネットワークとして記述する。これにより、人や物の流れを表現・分析可能となり、公共交通網の見直しや公共施設の再配置、ナビゲーション等に利用することを想定している。LOD1では道路を面として記述する。これにより道路がもつ空間機能の表現・分析が可能となり、火災の延焼シミュレーションや埋設物管理といったシミュレーションや施設管理に利用できる。LOD2及びLOD3では道路を3次元で記述し、かつ横断構成要素を詳細に区分できるため、バリアフリーナビゲーションや自動運転といった詳細なデータを要求するユースケースに利用できる。


[[tocD_02_02]]
==== 使用可能な地物型とLOD

交通（道路）モデルは、LODごとに、使用すべき地物型やその空間属性が異なる。


[requirement]
.道路記述のLOD対応条件
====
[%metadata]
identifier:: /req/tran/1
subject:: 妥当な交通（道路）データセットの要件
description:: 道路の記述には、指定されたLODに対応する地物型及び空間属性を使用する。
====

道路の記述には、tran:Roadを使用する。CityGML3.0において、tran:Roadは、道路空間を表す。 +
この道路空間は、交通のために使用される交通空間（tran:TrafficSpace）、交通のために使用されない空間（tran:AuxiliaryTrafficSpace）に分かれる。また、交通空間上空の障害物のない空間（tran:ClearanceSpace）に分けることもできる。

tran:TrafficSpace及びtran:AuxiliaryTrafficSpaceが表す空間の底面は、路面となる。それぞれに対応する路面は、交通領域（tran:TrafficArea）及び交通補助領域（tran:AuxiliaryTrafficArea）である。

また、これらの路面上に存在するマンホールの蓋や損傷は、穴の面（tran:HoleSurface）として区別することができる。

さらに、複数の空間を、区間（tran:Section）や交差点（tran:Intersection）としてグループ化できる。

道路の記述に使用する各地物型は、空間属性を持つものと持たないものがある。空間属性を持つ場合、LODに応じた空間属性を持つ。

* LOD0では、tran:Roadに含まれるtran:TrafficSpaceの境界となるtran:TrafficAreaを使用し、道路の形状を線で記述する。
* LOD1では、tran:Roadに含まれるtran:TrafficSpaceの境界となるtran:TrafficAreaを使用し、道路の形状を面で記述する。また、必要に応じてtran:AuxiliaryTrafficSpaceの境界となるtran:AuxiliaryTrafficAreaを区分することができる。
* LOD2では、tran:Roadに含まれるtran:TrafficSpaceの境界となるtran:TrafficArea及びtran:AuxiliaryTrafficSpaceの境界となるtran:AuxiliaryTrafficAreaを使用し、道路の形状を、高さを持つ面で記述する。
* LOD3では、tran:Roadに含まれるtran:TrafficSpaceの境界となるtran:TrafficArea及びtran:AuxiliaryTrafficSpaceの境界となるtran:AuxiliaryTrafficAreaを使用し、道路の形状を、高さを持つ面で記述する。このとき、LOD2よりもさらに詳細な形状が表現される。

tran:Roadは空間属性を持たず地物のコンテナとして機能する。このとき、道路の形状はこれに含まれる地物の空間属性により表現する。

各LODでは、tran:Section及びtran:Intersectionを使用できる。これらは、tran:TrafficSpace及びtran:AuxiliaryTrafficSpaceの集まりとなる。tran:Section（区間）及びtran:Intersectionも空間属性は持たない。各LODにおいて、tran:TrafficSpace及びtran:AuxiliaryTrafficSpaceは、空間属性を省略できる。

さらに、各LODでは、路面上に存在するマンホールの蓋や損傷を表す範囲をtran:HoleSurfaceとして記述することができる。

==== 交通（道路）モデルのLOD定義

交通（道路）モデルは、LODごとに、含むべき地物やその取得基準を定めている。

[[tocD_02_03_01]]
===== LOD0

交通（道路）モデル（LOD0）は、道路の形状を、高さを持たない中心線として記述する。中心線の記述には、tran:TrafficAreaの空間属性（core:lod0MultiCurve）を使用する。 +
このとき、中心線は、道路、上り線と下り線、車線、歩道のいずれかとなる。 +
中心線の粒度は、<<tab-D-2>>に示す主題属性を用いて区分する。

| 上り線、下り線の中心線 | way | 車道部
| 車線の中心線 | lane | 車線

[[tocD_02_03_02]]
===== LOD1

交通（道路）モデル（LOD1）は、道路の形状を、高さを持たない多曲面として記述する。 +
多曲面の記述には、tran:TrafficAreaの空間属性（core:lod1MultiSurface）を使用する。 +
このとき、多曲面は道路全幅、上り線/下り線、車線又は歩道のいずれかとなる。

[[tocD_02_03_03]]
===== LOD2

交通（道路）モデル（LOD2）は、道路の形状を高さ、を持つ多曲面として記述する。 +
多曲面の記述には、tran:TrafficAreaの空間属性（core:lod2MultiSurface）及びtran:AuxiliaryTrafficAreaの空間属性（core:lod2MultiSurface）を使用する。

交通（道路）モデル（LOD2）は、道路内の区分と高さの取得方法により、LOD2.0、LOD2.1及びLOD2.2に細分される。<<tab-D-5>>及び<<tab-D-6>>に、細分したLOD2の概要を示す。

// 交通（道路）モデルにおける属性granularityによる「道路内の区分」
[[tab-D-5]]
include::../../../common/sections/com_road_Category_lod2.adoc[]

// 交通（道路）モデルにおけるLOD2.0、LOD2.1及びLOD2.2の「高さの取得方法」
[[tab-D-6]]
include::../../../common/sections/com_road_Category_height_lod2.adoc[]

LOD1では交通（道路）オブジェクトは高さを持たないため、立体交差や歩道と車道との間の高低差を表現できない。これらが必要な場合には、高さを持つLOD2又はLOD3が必要となる。

// 交通（道路）モデル（LOD2）の取得イメージ（道路内の区分）
[[tab-D-7]]
include::../../../common/sections/com_road_Acquire_lod2_category.adoc[]

LOD2において最も粗い「道路内の区分」では、道路を車道部、車道交差部、島及び歩道部に区分する。この区分はLOD2.0に適用され、この場合、tran:TrafficSpaceの属性tran:granularityの値は、“way”となる。

また、LOD2.0において、車線を区分する場合、tran:TrafficSpaceの属性tran:granularityの値は、“lane”となる。LOD2.1では、LOD2.0の区分に加え、歩道部のうち歩道上の植栽を区分する。すなわち、歩道においても、通行の用に供される部分を明確に区分できる。このとき、車道部も車線に区分され、tran:TrafficSpaceの属性tran:granularityの値は、“lane”となる。

LOD2.2では、道路の横断構成要素をさらに細分する。車道部のうち、路肩や停車帯等を区別したり、歩道を歩道や自転車歩行車道等に区別したりできる。LOD2.2における横断構成要素の取得基準の区分はユースケースごとに決定できる。

// 交通（道路）モデル（LOD2）の取得イメージ（高さの取得方法）
[[tab-D-8]]
include::../../../common/sections/com_road_Acquire_lod2_height.adoc[]

LOD2.0の「高さの表現」では、高架橋、立体交差等、道路全体の上下の階層構造を区別できる。

LOD2.1では、道路内の縁石による高さの差を表現する。この高さの差は、縁石によりマウントアップされた歩道と車道との高さの差や車道内の分離帯や交通島の高さの差（概ね15㎝程度）とする。横断歩道への接続や車両の出入り等の目的で歩道に設けられた、歩道の切り下げ部では緩やかに車道の高さに擦り付ける（歩道切り下げ部と車道との間に生じる高さの差の表現は行わない）。

さらに、LOD2.2では、歩道切り下げ部と車道との高さの差（概ね2㎝程度）の表現を行う。なお、LOD2.2における高さの取得基準の下限値はユースケースにより決定できる。

なお、LOD2.0及びLOD2.1は、航空写真等上空から取得したデータの利用を前提とした区分である。このとき、トンネル内や高架橋の下部等の遮蔽部は上空から取得したデータでは作成することができない。そのため、他の資料による補完又は、推定によるデータ作成をする必要がある。LOD2.2は、MMS（Mobile Mapping System）により取得した点群や画像等のデータの利用を前提とした区分である。

データ集合に、航空写真等による図化、他の資料による補完、推定による作図というように、作成方法が異なる交通（道路）オブジェクトが混在する場合は、データ品質属性（<<tocD_03_02_08>>）を使用し、個々の交通（道路）オブジェクトの品質情報を記述することで、それぞれの作成方法を明示できる。


[[tocD_02_03_04]]
===== LOD3

交通（道路）モデル（LOD3）は、道路の形状を、高さを持つ多曲面として記述する。

多曲面の記述には、tran:TrafficAreaの空間属性（core:lod3MultiSurface）及びtran:AuxiliaryTrafficAreaの空間属性（core:lod3MultiSurface）を使用する。

LOD3は、道路の形状をさらに詳細に表現することができる。例えば、道路の横断構成要素をさらに細分したり、道路の表面の凹凸を表現したりできる。

LOD3は、道路の詳細設計等で作成される3次元モデル成果物の利用を想定した区分である。

[[tocD_02_04]]
==== 3D都市モデルに含むべき道路のLOD


[requirement]
.道路3Dモデルの幾何オブジェクト要件
====
[%metadata]
identifier:: /req/tran/2
subject:: 妥当な交通（道路）データセットの要件
description:: 道路の3D都市モデルには、LOD1の幾何オブジェクトを必ず含む。
====

3D都市モデルに道路を含む場合には、幾何オブジェクトとして、LOD1を必ず記述しなければならない。また、ユースケースの必要に応じ、LOD0、LOD2又はLOD3を記述することができる。
